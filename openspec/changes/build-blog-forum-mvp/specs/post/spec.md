## 新增需求

### 需求:创建和查看草稿
系统必须允许注册用户创建并查看自己的文章草稿，同时禁止访客或其他用户访问该草稿。

#### 场景:创建草稿成功
- **当** 已认证用户提交合法的文章标题和 Markdown 正文创建草稿
- **则** 系统创建归属于该用户的草稿
- **并且** 该草稿不出现在任何公开文章列表中

#### 场景:草稿参数非法
- **当** 已认证用户提交空标题、超长内容或其他非法文章参数
- **则** 系统拒绝创建并返回参数错误
- **并且** 系统不保存部分草稿

#### 场景:访问他人草稿
- **当** 用户或访客请求不属于自己的草稿
- **则** 系统返回数据不存在
- **并且** 系统不泄露草稿作者、标题或正文

### 需求:编辑文章
系统必须允许作者编辑自己的草稿或已发布文章，发布后的合法编辑直接生效且不触发重新审核，并拒绝越权、非法和并发冲突的更新。

#### 场景:编辑自己的文章
- **当** 作者以最新版本提交合法标题、正文、分类和标签
- **则** 系统更新文章内容
- **并且** 已发布文章保持已发布状态并直接展示更新内容

#### 场景:编辑他人文章
- **当** 注册用户尝试编辑其他作者的文章
- **则** 系统拒绝操作并返回无权限或数据不存在结果
- **并且** 文章内容保持不变

#### 场景:编辑不存在文章
- **当** 作者请求编辑不存在或已删除的文章
- **则** 系统返回数据不存在或状态冲突
- **并且** 系统不创建替代文章

#### 场景:并发编辑冲突
- **当** 两个请求基于同一旧版本并发编辑同一篇文章
- **则** 系统最多接受一个更新
- **并且** 其他请求返回状态冲突而不得覆盖已成功的更新

### 需求:发布文章
系统必须允许作者将合法草稿发布为公开文章，并对重复发布、缺失必要内容和无效分类返回可预测结果。

#### 场景:发布成功
- **当** 作者发布内容完整、分类有效且当前为草稿状态的文章
- **则** 系统将文章转为已发布状态并记录发布时间
- **并且** 文章可以通过公开 slug 和公开列表访问

#### 场景:重复发布
- **当** 作者再次发布已经处于已发布状态的文章
- **则** 系统返回文章已发布的幂等结果或明确状态结果
- **并且** 系统不重复生成文章或改变首次发布时间

#### 场景:发布条件不满足
- **当** 草稿缺少必需内容、引用无效分类或已被删除
- **则** 系统拒绝发布并返回参数错误、数据不存在或状态冲突
- **并且** 文章不会进入公开列表

### 需求:稳定文章 slug
系统必须为公开文章提供全局唯一且稳定的 slug，标题编辑不得自动改变既有 slug，已使用的 slug 不得被其他文章复用。

#### 场景:生成唯一 slug
- **当** 作者首次发布一篇尚无公开 slug 的文章
- **则** 系统为该文章分配可用于公开访问的唯一 slug
- **并且** slug 冲突不会导致两篇文章共享同一公开地址

#### 场景:编辑标题
- **当** 作者修改已发布文章的标题
- **则** 系统保留文章原有 slug
- **并且** 原公开地址继续指向同一篇文章

#### 场景:访问不存在 slug
- **当** 访客使用不存在或从未分配的 slug 查询文章
- **则** 系统返回数据不存在
- **并且** 系统不返回相似 slug 对应的其他文章

### 需求:Markdown 内容安全
系统必须保存作者提交的 Markdown 语义，并确保面向公开展示的内容不包含可执行脚本或其他危险内容。

#### 场景:合法 Markdown
- **当** 作者提交符合长度和内容限制的 Markdown
- **则** 系统接受并在公开展示时保留允许的格式
- **并且** 展示结果不执行作者提交的代码或事件处理逻辑

#### 场景:危险 Markdown
- **当** 作者提交包含脚本、危险链接协议或不允许嵌入内容的 Markdown
- **则** 系统拒绝内容或移除危险部分
- **并且** 任何公开响应均不得包含可执行危险内容

#### 场景:Markdown 参数超限
- **当** 作者提交空正文或超过允许大小的 Markdown
- **则** 系统拒绝保存并返回参数错误
- **并且** 原文章内容保持不变

### 需求:作者删除文章
系统必须允许作者软删除自己的文章，删除操作保持幂等，并禁止其他用户删除该文章。

#### 场景:删除成功
- **当** 作者请求删除自己的草稿或已发布文章
- **则** 系统将文章标记为已删除
- **并且** 文章不再出现在公开查询、评论入口或点赞入口中

#### 场景:重复删除
- **当** 作者再次删除已经由自己删除的文章
- **则** 系统返回与删除完成一致的结果
- **并且** 系统不产生新的文章状态或重复副作用

#### 场景:删除他人文章
- **当** 注册用户请求删除其他作者的文章
- **则** 系统拒绝操作并返回无权限或数据不存在结果
- **并且** 目标文章保持不变

### 需求:公开文章查询
系统必须只向访客展示已发布、未隐藏且未删除的文章，并支持按分类和标签筛选、分页及稳定排序。

#### 场景:默认文章列表
- **当** 访客未提交筛选、分页和排序参数浏览文章
- **则** 系统按发布时间倒序返回第一页且默认最多 20 条
- **并且** 结果仅包含公开可见文章并保持稳定顺序

#### 场景:按分类和标签筛选
- **当** 访客提交存在的分类或标签筛选条件
- **则** 系统仅返回同时满足筛选条件的公开文章
- **并且** 分页总数基于筛选后的结果计算

#### 场景:合法分页和排序
- **当** 访客提交合法页码、每页数量和允许的排序方式
- **则** 系统返回对应范围和顺序的公开文章
- **并且** 每页数量不超过 100 条

#### 场景:查询参数非法
- **当** 访客提交非法分页值、不支持的排序字段或无效筛选标识
- **则** 系统拒绝请求并返回参数错误或数据不存在
- **并且** 系统不执行不受支持的排序或筛选表达式

#### 场景:访问非公开文章
- **当** 普通访客通过 slug 请求草稿、隐藏或已删除文章
- **则** 系统返回数据不存在
- **并且** 系统不泄露文章的实际状态和内容

### 需求:分类管理
系统必须只允许管理员创建、修改、停用和查询分类，并阻止普通用户修改分类。

#### 场景:管理员创建分类
- **当** 管理员提交合法且未重复的分类名称
- **则** 系统创建可供文章选择的分类
- **并且** 新分类可用于公开文章筛选

#### 场景:普通用户管理分类
- **当** 非管理员尝试创建、修改或停用分类
- **则** 系统拒绝操作并返回无权限结果
- **并且** 分类状态保持不变

#### 场景:分类参数冲突
- **当** 管理员提交空名称、重复名称或请求修改不存在的分类
- **则** 系统返回参数错误、冲突或数据不存在
- **并且** 系统不创建重复分类

#### 场景:停用已使用分类
- **当** 管理员停用已被文章使用的分类
- **则** 系统禁止新文章继续选择该分类
- **并且** 已发布文章仍保留原分类信息并可被读取

### 需求:标签自动创建
系统必须允许作者在保存文章时自由提供合法标签，并自动复用已有标签或创建新标签，且并发请求不得产生语义重复标签。

#### 场景:自动创建新标签
- **当** 作者保存文章时提交合法且当前不存在的标签
- **则** 系统创建标签并关联到文章
- **并且** 标签可以用于公开文章筛选

#### 场景:复用已有标签
- **当** 作者提交与已有标签规范化后相同的名称
- **则** 系统复用已有标签
- **并且** 系统不创建大小写或空白差异导致的重复标签

#### 场景:并发创建相同标签
- **当** 多个请求并发提交规范化后相同的新标签
- **则** 系统最终只保留一个可用标签身份
- **并且** 所有成功文章均关联到同一标签

#### 场景:标签参数非法
- **当** 作者提交空标签、超长标签或超过单篇文章允许数量的标签
- **则** 系统拒绝请求并返回参数错误
- **并且** 不保存不完整的文章标签关系
