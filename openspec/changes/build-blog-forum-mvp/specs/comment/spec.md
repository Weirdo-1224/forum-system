## 新增需求

### 需求:发表评论
系统必须允许注册用户在公开可见文章下发表评论，并拒绝访客、非法内容和不可评论文章上的请求。

#### 场景:发表评论成功
- **当** 已认证用户在已发布且可见的文章下提交合法评论
- **则** 系统创建归属于该用户和文章的一级评论
- **并且** 新评论可以出现在该文章的公开评论列表中

#### 场景:访客发表评论
- **当** 未认证访客尝试发表评论
- **则** 系统拒绝请求并返回认证失败
- **并且** 系统不创建评论

#### 场景:评论内容非法
- **当** 用户提交空内容、超长内容或不允许的内容
- **则** 系统拒绝请求并返回参数错误
- **并且** 系统不保存部分评论

#### 场景:文章不存在或不可评论
- **当** 用户对不存在、草稿、隐藏或已删除文章发表评论
- **则** 系统返回数据不存在或状态冲突
- **并且** 系统不泄露不可见文章的详细状态

### 需求:两层回复
系统必须支持最多两层的评论展示；回复二级评论时仍归入同一一级评论之下，并保留被回复目标。

#### 场景:回复一级评论
- **当** 已认证用户对公开文章下存在的一级评论提交合法回复
- **则** 系统创建该一级评论下的二级回复
- **并且** 公开展示不产生第三层结构

#### 场景:回复二级评论
- **当** 已认证用户对存在的二级回复提交合法回复
- **则** 系统将新回复展示在同一一级评论的二级列表中
- **并且** 系统保留本次回复所针对的二级评论信息

#### 场景:父评论不存在或不可用
- **当** 用户回复不存在、已隐藏或不属于目标文章的评论
- **则** 系统拒绝请求并返回数据不存在或状态冲突
- **并且** 系统不创建脱离目标文章的回复

### 需求:编辑自己的评论
系统必须允许评论作者编辑自己尚未删除或隐藏的评论，并拒绝越权、非法参数和状态冲突。

#### 场景:编辑成功
- **当** 评论作者提交合法的新内容编辑自己的可编辑评论
- **则** 系统更新评论内容并保留其原有层级关系
- **并且** 后续公开查询返回更新后的内容

#### 场景:编辑他人评论
- **当** 注册用户尝试编辑其他用户的评论
- **则** 系统拒绝操作并返回无权限或数据不存在结果
- **并且** 评论内容保持不变

#### 场景:编辑已删除或隐藏评论
- **当** 作者尝试编辑已经删除或被管理员隐藏的评论
- **则** 系统拒绝操作并返回状态冲突
- **并且** 评论状态和内容保持不变

#### 场景:编辑参数非法
- **当** 作者提交空内容或超过长度限制的内容
- **则** 系统拒绝编辑并返回参数错误
- **并且** 原评论内容保持不变

### 需求:删除评论并保留子回复
系统必须允许评论作者软删除自己的评论；删除一级评论时保留其二级回复并将这些回复标记为 orphan，重复删除保持幂等。

#### 场景:删除无回复评论
- **当** 作者删除自己没有子回复的评论
- **则** 系统将该评论标记为已删除
- **并且** 公开查询不再展示原评论正文

#### 场景:删除有回复的一级评论
- **当** 作者删除自己仍有二级回复的一级评论
- **则** 系统隐藏父评论原正文并保留全部二级回复
- **并且** 系统将保留的二级回复标记为 orphan

#### 场景:删除与新增回复并发
- **当** 删除一级评论和新增其回复的请求并发发生
- **则** 系统不得产生指向不可识别父评论的普通回复
- **并且** 新回复要么被拒绝，要么与其他子回复一致地标记为 orphan

#### 场景:重复删除
- **当** 作者再次删除已经删除的评论
- **则** 系统返回与删除完成一致的结果
- **并且** 已保留子回复及 orphan 状态不被重复破坏

#### 场景:删除他人评论
- **当** 注册用户尝试删除其他用户的评论
- **则** 系统拒绝操作并返回无权限或数据不存在结果
- **并且** 目标评论及回复保持不变

### 需求:公开评论分页与排序
系统必须按稳定顺序分页返回文章的公开评论，并保持两层展示关系和 orphan 语义。

#### 场景:默认查询
- **当** 访客未提交分页和排序参数查看公开文章评论
- **则** 系统按允许的默认顺序返回一级评论且默认最多 20 条
- **并且** 每个一级评论携带其允许展示的二级回复

#### 场景:合法分页和排序
- **当** 访客提交合法页码、每页数量和允许的时间排序方式
- **则** 系统返回对应范围且顺序稳定的一级评论
- **并且** 每页一级评论数量不超过 100 条

#### 场景:非法分页或排序
- **当** 访客提交负数页码、超过上限的每页数量或不支持的排序字段
- **则** 系统拒绝请求并返回参数错误
- **并且** 系统不执行不受支持的排序表达式

#### 场景:展示 orphan 回复
- **当** 公开评论结果包含父评论已删除但子回复保留的评论组
- **则** 系统以删除占位语义展示父级位置并返回允许公开的 orphan 回复
- **并且** 系统不返回已删除父评论的原正文

### 需求:评论可见性
系统必须对隐藏或删除评论实施一致的可见性规则，并在文章本身不可见时禁止公开读取其评论。

#### 场景:普通用户查看隐藏评论
- **当** 访客或非作者请求被管理员隐藏的评论
- **则** 系统不返回该评论正文
- **并且** 系统不向其泄露具体审核状态

#### 场景:文章被隐藏或删除
- **当** 访客请求已隐藏或已删除文章的评论列表
- **则** 系统返回文章数据不存在
- **并且** 系统不单独暴露该文章下的评论
