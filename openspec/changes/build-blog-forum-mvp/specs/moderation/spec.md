## 新增需求

### 需求:隐藏和恢复文章
系统必须只允许管理员隐藏或恢复文章，隐藏操作必须记录合法原因，并且不得通过恢复操作重新公开作者已删除的文章。

#### 场景:隐藏文章成功
- **当** 管理员为存在且未删除的文章提交合法审核原因执行隐藏
- **则** 系统将文章设为对公众不可见
- **并且** 系统记录本次审核操作

#### 场景:重复隐藏文章
- **当** 管理员再次隐藏已经隐藏的同一文章
- **则** 系统返回与文章已隐藏一致的幂等结果
- **并且** 系统不重复改变文章状态

#### 场景:恢复文章成功
- **当** 管理员恢复由审核操作隐藏且作者未删除的文章
- **则** 系统撤销其隐藏状态
- **并且** 已发布文章重新对公众可见并记录恢复操作

#### 场景:恢复作者已删除文章
- **当** 管理员尝试恢复已被作者删除的文章
- **则** 系统拒绝操作并返回状态冲突
- **并且** 文章保持不可见且不得被重新公开

#### 场景:审核原因非法
- **当** 管理员提交空原因或超过允许长度的原因隐藏文章
- **则** 系统拒绝操作并返回参数错误
- **并且** 文章状态和审核记录保持不变

#### 场景:非管理员审核文章
- **当** 普通用户尝试隐藏或恢复文章
- **则** 系统拒绝操作并返回无权限结果
- **并且** 文章状态保持不变

#### 场景:文章不存在
- **当** 管理员请求隐藏或恢复不存在的文章
- **则** 系统返回数据不存在
- **并且** 系统不创建无目标的审核记录

### 需求:隐藏和恢复评论
系统必须只允许管理员隐藏或恢复评论，隐藏操作必须记录合法原因，并且不得恢复作者已删除的评论正文。

#### 场景:隐藏评论成功
- **当** 管理员为存在且未删除的评论提交合法原因执行隐藏
- **则** 系统将评论设为对公众不可见
- **并且** 系统记录本次审核操作且不自动隐藏整篇文章

#### 场景:恢复评论成功
- **当** 管理员恢复由审核操作隐藏且作者未删除的评论
- **则** 系统撤销评论隐藏状态
- **并且** 评论在其文章仍公开时重新可见并记录恢复操作

#### 场景:恢复已删除评论
- **当** 管理员尝试恢复作者已删除的评论
- **则** 系统拒绝操作并返回状态冲突
- **并且** 评论原正文保持不可见

#### 场景:非管理员审核评论
- **当** 普通用户尝试隐藏或恢复评论
- **则** 系统拒绝操作并返回无权限结果
- **并且** 评论状态保持不变

#### 场景:评论不存在或参数非法
- **当** 管理员请求审核不存在评论或提交非法审核原因
- **则** 系统返回数据不存在或参数错误
- **并且** 系统不创建无效审核记录

### 需求:隐藏内容的访问边界
系统必须允许作者查看自己被隐藏内容的完整内容、隐藏状态和审核原因，同时对其他普通访问者表现为数据不存在。

#### 场景:作者查看隐藏文章
- **当** 已认证作者查看自己被管理员隐藏的文章
- **则** 系统返回该文章内容及隐藏状态
- **并且** 系统返回作者可见的审核原因

#### 场景:作者查看隐藏评论
- **当** 已认证作者查看自己被管理员隐藏的评论
- **则** 系统返回该评论内容及隐藏状态
- **并且** 系统返回作者可见的审核原因

#### 场景:其他用户查看隐藏内容
- **当** 访客或其他普通用户请求被隐藏的文章或评论
- **则** 系统返回数据不存在
- **并且** 系统不泄露内容是否因审核被隐藏

### 需求:禁用和恢复用户
系统必须只允许管理员禁用或恢复用户；禁用后阻止新登录和 Token 刷新，但历史公开内容继续展示。

#### 场景:禁用用户成功
- **当** 管理员禁用一个存在且当前有效的注册用户
- **则** 系统将该用户设为禁用状态并阻止后续登录和 Token 刷新
- **并且** 该用户既有的公开文章、评论和公开主页继续展示

#### 场景:重复禁用用户
- **当** 管理员再次禁用已经禁用的用户
- **则** 系统返回与用户已禁用一致的幂等结果
- **并且** 用户历史内容和账号状态不产生额外变化

#### 场景:恢复用户成功
- **当** 管理员恢复一个被禁用的用户
- **则** 系统允许该用户重新使用正确凭据登录
- **并且** 系统不恢复禁用前已撤销或过期的 Token

#### 场景:普通用户管理账号状态
- **当** 非管理员尝试禁用或恢复用户
- **则** 系统拒绝操作并返回无权限结果
- **并且** 目标用户状态保持不变

#### 场景:目标用户不存在
- **当** 管理员请求禁用或恢复不存在的用户
- **则** 系统返回数据不存在
- **并且** 系统不创建无目标的审核记录

### 需求:审核操作记录
系统必须为文章隐藏恢复、评论隐藏恢复以及用户禁用恢复保留不可被普通用户修改的审核操作记录。

#### 场景:记录审核操作
- **当** 管理员成功完成内容或用户审核操作
- **则** 系统记录操作者、目标、动作、原因和发生时间
- **并且** 后续状态变化不得覆盖既有记录

#### 场景:管理员分页查看记录
- **当** 管理员使用合法分页、筛选和排序参数查看审核记录
- **则** 系统返回符合条件的记录且默认每页最多 20 条
- **并且** 每页数量不超过 100 条并保持稳定排序

#### 场景:审核记录查询参数非法
- **当** 管理员提交非法分页值、不支持的排序字段或无效筛选条件
- **则** 系统拒绝请求并返回参数错误
- **并且** 系统不执行不支持的查询表达式

#### 场景:普通用户查看审核记录
- **当** 非管理员尝试查看完整审核操作记录
- **则** 系统拒绝访问并返回无权限结果
- **并且** 系统不泄露其他用户或管理员的审核信息

#### 场景:删除或修改审核记录
- **当** 任意用户尝试通过业务接口删除或修改既有审核记录
- **则** 系统拒绝该操作
- **并且** 既有审核历史保持不变
