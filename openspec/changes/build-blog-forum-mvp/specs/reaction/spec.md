## 新增需求

### 需求:文章点赞
系统必须允许注册用户对公开可见文章点赞，并保证重复请求和并发请求不会产生重复点赞。

#### 场景:首次点赞
- **当** 已认证用户对尚未点赞的公开文章请求点赞
- **则** 系统记录该用户对文章的点赞
- **并且** 文章点赞数增加一次

#### 场景:重复点赞
- **当** 用户再次对已经点赞的同一文章请求点赞
- **则** 系统返回与点赞完成一致的幂等结果
- **并且** 点赞记录和点赞数不重复增加

#### 场景:并发点赞
- **当** 同一用户并发多次点赞同一篇文章
- **则** 系统最终只保留一次有效点赞
- **并且** 文章点赞数只增加一次

#### 场景:访客点赞
- **当** 未认证访客请求点赞文章
- **则** 系统拒绝请求并返回认证失败
- **并且** 点赞状态和数量保持不变

#### 场景:文章不存在或不可点赞
- **当** 用户点赞不存在、草稿、隐藏或已删除文章
- **则** 系统返回数据不存在或状态冲突
- **并且** 系统不创建点赞

### 需求:取消文章点赞
系统必须允许注册用户取消自己的文章点赞，且对未点赞文章的取消请求保持幂等。

#### 场景:取消成功
- **当** 用户对自己已经点赞的文章请求取消点赞
- **则** 系统移除该用户的有效点赞
- **并且** 文章点赞数减少一次且不得小于零

#### 场景:重复取消
- **当** 用户对自己当前未点赞的文章请求取消点赞
- **则** 系统返回与取消完成一致的幂等结果
- **并且** 点赞数保持不变

#### 场景:点赞与取消并发
- **当** 同一用户对同一文章并发发起点赞和取消点赞
- **则** 系统以最终成功完成的状态作为用户当前点赞状态
- **并且** 点赞记录与公开点赞数保持一致

### 需求:查询点赞状态与数量
系统必须返回文章当前有效点赞总数，并为已认证用户返回其个人点赞状态。

#### 场景:访客查看点赞数
- **当** 访客查看公开文章
- **则** 系统返回该文章当前有效点赞总数
- **并且** 系统不要求访客具有个人点赞状态

#### 场景:用户查看个人状态
- **当** 已认证用户查看公开文章
- **则** 系统返回文章有效点赞总数
- **并且** 系统同时表明当前用户是否已点赞

#### 场景:高并发后的计数一致性
- **当** 多个用户并发点赞或取消点赞同一篇文章并全部处理完成
- **则** 系统返回的点赞总数与当前有效点赞关系数量一致
- **并且** 重复请求不会导致计数偏大、偏小或出现负数

### 需求:点赞范围限制
系统必须将 MVP 点赞能力限制为文章，不得接受评论、用户或其他对象的点赞请求。

#### 场景:请求点赞非文章对象
- **当** 用户尝试对评论、用户或其他非文章对象点赞
- **则** 系统拒绝请求并返回参数错误或不支持的操作
- **并且** 系统不创建任何点赞状态
